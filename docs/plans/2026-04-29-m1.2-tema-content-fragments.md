**Goal:** Site Hotel Livingstone com **identidade visual própria** rodando nas 6 pages (PT-BR + EN-US), composto por:
- **Tema custom** `hotel-livingstone-theme` (parent Styled, criado via `blade create -t js-theme`) — header, footer, paleta, tipografia, logo
- **3 Web Content Articles "Quarto"** (Suíte Kilimanjaro, Serengeti, Okavango) cadastrados via UI
- **3 fragments custom** (Hero, Card de Quarto **mapeado** pro Article, Galeria) na Collection "Hotel Livingstone Components"
- **6 pages compostas** com fragments + mapping + tema novo
- **Tradução EN-US** dos display names das 6 pages (pendência herdada do M1.1)
- **3 ADRs novos (002-004)** + retrospectiva M1.2 + tag git `m1.2-done`

**Architecture:** Workspace nativo (porta 8081) — mesmo padrão do M1.1. Tema fica versionado em `workspace/themes/hotel-livingstone-theme/`. Web Content (Structure, Template, Articles) e Fragments ficam **no DB do Liferay, não no git** (mesma limitação do M1.1; Site Initializer entra no M2). CSS tokens fixados antes de fragments — fragments **consomem** tokens, não duplicam.

**Tech Stack:** Liferay 7.4 CE GA132, Blade CLI 8.0.0, Java 21 (Liberica), Gradle (via wrapper), Yeoman + generator-liferay-theme (chamado pelo Blade automaticamente), Node + npm (workspace baixa em `node-x.y.z/` se faltar), FreeMarker (FTL), SCSS, JavaScript ES6+, Markdown, Git.

## Decisões de design (resumo da brainstorm 2026-04-29)

Brainstorm completo no histórico da sessão. Decisões fechadas:

| # | Decisão | Escolha | Alternativa descartada |
|---|---|---|---|
| **0** (reframe ADR-001 do spec) | Tema custom vs Fragment Collection | **Os DOIS, com escopos definidos** — tema cobre estrutura/casca global; fragments cobrem blocos reutilizáveis | "Ou um ou outro" — fere quem-edita-mora |
| **1** | Template do tema | **`blade create -t js-theme`** (caminho oficial 7.4 — Blade chama Yeoman por baixo) | `theme` (legacy, possivelmente removido em GA132); `theme-contributor` (sem FTL, fora de escopo) |
| **2** | Parent theme | **Styled** (CSS base + FTL skeleton mínimo; identidade própria) | Classic (fica "Liferay vanilla com cores"); Unstyled (custo fora do orçamento M1) |
| **3** | Ordem de execução | **A — Top-down** (Tema → WC → Fragments → Composição → Polimento) | B (bottom-up: WC sem tema é "campo abstrato"); C (vertical slice: caótico pra Q&A guiado) |

**Regra mestra cravada na brainstorm:** *"quem edita = onde mora; estrutura ≠ conteúdo"*. Aplicada em ADR-002.

**Mapa de camadas do M1.2:**

| Camada | Mecanismo | Cobre | Quem edita |
|---|---|---|---|
| Casca global | Tema custom | header skeleton, footer skeleton, paleta, tipografia, font stack | Dev (código) |
| Bloco apresentacional | Fragment custom | Card, Hero, Galeria — *como mostrar* | Content creator (Page Editor) |
| Dado estruturado | Web Content Structure + Article | Quarto — *o que mostrar* | Content creator (Web Content) |
| Conteúdo de navegação | Navigation Menu + NavItems | Menu Principal (Task 8 do M1.1) | Content creator (Site Builder) |
